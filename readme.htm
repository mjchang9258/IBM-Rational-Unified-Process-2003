This file provides troubleshooting information when running Rational Unified Process and Extended Help in the Mckinley Beta Release.

  1. JRE plugin:
    We recommend that you use the Sun Java 1.3.1_0X plugin. If you do not already have the plugin installed, you will be prompted to do so as soon as you try launching RUP or Extended Help for the first time.

  2. If running RUP or Extended Help from an HTTP server and you receive a message informing you that limited read/write access is required:
    Locate the java.policy file in your installed Sun Java plugin directory (e.g. C:\Program Files\JavaSoft\JRE\1.3.1_02\lib\security).
    Add the following permissions to the java.policy file (add them after the last "permissions" line and before the "};" tag at the end of the file):
            permission java.io.FilePermission "C:\\Documents and Settings\\username\\MyRup", "read,write,delete";
            permission java.io.FilePermission "C:\\Documents and Settings\\username\\MyRup\\-", "read,write,delete";
            permission java.lang.RuntimePermission "loadLibrary.environment";
            permission java.util.PropertyPermission "user.name", "read";
            permission java.util.PropertyPermission "user.home", "read";
            permission java.util.PropertyPermission "user.dir", "read";
            permission java.util.PropertyPermission "file.encoding", "read";
            permission java.awt.AWTPermission "accessClipboard";
            permission java.awt.AWTPermission "showWindowWithoutWarningBanner";

    Change the string C:\\Documents and Settings\\username\\MyRup to match with your user profile.
    For example, your string might look like C:\\Documents and Settings\\jdoe\\MyRup.
    Notice that the change is to username in the string.

    Also, change the string C:\\Documents and Settings\\username\\MyRup\\- to match with your user profile.
    For example, your string might look like C:\\Documents and Settings\\jdoe\\MyRup\\-.
    Notice that the change is to username in the string.

    TIP: to find what your user profile is, run the command echo %USERPROFILE% in a command prompt window.

  3. When running a locally installed RUP or Extended Help and you receive a message informing you that limited read/write access is required:
    Locate the directory where you had installed RUP. Go up a level to the Common directory and
    run ConfigurePolicy.bat. Typically, this is C:\Program Files\Rational\Common\ConfigurePolicy.bat.

  4. Make sure all your browser windows are closed before restarting RUP again.
    To start RUP locally, go to Start -> Program Files -> Rational Unified Process (McKinley Beta) -> Rational Unified Process (McKinley Beta).
    To start RUP from an HTTP server, simply launch the appropriate URL again.
